## 1. このプロダクトが満たすべきこと（要件視点での一言）

チームがスプリントごとにふりかえり（KPT）を行い、**AIが「そのアクションは本当に再発防止になるか／形骸化してないか」を問い返す**。
出したアクションは**次スプリントで効果を検証**し、効いていなければ追い続ける。＝**改善ループを閉じる**。

MVPで「動くループ」を最小構成で1周見せられることがゴール。

---

## 2. アクター（登場人物）

MVPでは役割を分けない（＝権限管理はやらない、BRIEF方針）。

| アクター | 説明 |
|---|---|
| **メンバー** | チームに属し、付箋を書き、AI問い返しを実行し、アクションの効果判定を付ける人。MVPでは全員同一権限。 |
| **AI（問い返し役）** | 付箋群を受け取り、改善アクション案＋問い返しを構造化で返す。人格は「問い返す先輩スクラムマスター」。 |

*招待フロー・複数人同時編集はMVP対象外（v1以降）。当面は1人〜少人数が同じチームを共有する前提。*

## 3. ドメイン用語（データモデルの種）

| 用語 | 意味 | 補足 |
|---|---|---|
| **チーム(Team)** | ふりかえりを行う単位。 | Team > Sprint の浅い階層。 |
| **スプリント(Sprint)** | ふりかえり1回に対応する期間の箱。 | ［要レビュー］**1スプリント＝ふりかえり1回**で固定。 |
| **付箋(Note)** | KPTの1項目。 | 種別 = Keep / Problem / Try。リスト表示。 |
| **AI問い返し(Reflection)** | 付箋群に対するAIの1回の応答。 | 改善アクション案＋問い返し文＋前回アクションの効果判定を構造化で保持。 |
| **アクション(Action)** | 次に向けた改善策。 | 効果追跡の主役。状態を持ち、スプリントをまたいで追跡。効果判定は **AIが自動で付ける**（人は上書き可）。 |
| **ナレッジ(Knowledge)** | これまでのアクションとその結果・問い返しの蓄積。 | チーム単位で貯め続け、毎回のAI問い返しの入力に渡す。過去の学びを踏まえて判定・問い返しさせるための材料。 |

## 4. ユーザーストーリー（優先度：★=MVP必須 / ☆=あれば）

- ★ メンバーとして、チームとスプリントを作成できる（次のふりかえりを始められるように）
- ★ メンバーとして、スプリントに Keep/Problem/Try の付箋を追加・編集・削除できる
- ★ メンバーとして、ボタン1つで付箋群をAIに渡し、**改善アクション案と問い返し**を受け取れる
- ★ メンバーとして、AIが出したアクション案を**採用/編集/却下**し、採用したものをこのスプリントのアクションとして保存できる
- ★ メンバーとして、次スプリントでAI問い返しを実行すると、**前スプリントのアクションの効果をAIが自動判定**（効いた/効いてない/未着手＋理由）してくれる。違えば自分で上書きできる
- ★ メンバーとして、これまでのアクション・判定・問い返しが**ナレッジとして蓄積**され、AIが過去の学びを踏まえて問い返してくれる
- ★ 訪問者として、ログインして自分のチームのふりかえりだけを見られる（最小認証）
- ☆ メンバーとして、AIが「効いてない」と判定したアクションを次スプリントへ繰り越すか打ち切るか選べる
- ☆ メンバーとして、過去スプリントのふりかえり一覧を振り返れる

## 5. 主要フロー（ハッピーパス）

### 5.1 ふりかえり1回の流れ
1. スプリントを開く（無ければ作成）
2. **KPTを書く**：Keep/Problem/Try の付箋を追加・編集・削除
3. **AIに問い返してもらう**：ボタン押下 → 当スプリントの付箋 ＋ 前スプリントのアクション ＋ これまでのナレッジ をAIへ。AIは次を構造化で返す：
   - (a) **前回アクションの効果を自動判定**（効いた / 効いてない / 未着手）＋ 判定理由。
     **付箋に効果を示す言及があればそこから判定。言及が無く判定できないアクションは、AIが「◯◯は効きましたか?」とユーザに問い返し**、ユーザの回答で判定を確定する
   - (b) 「その再発防止、形骸化してない?」の**問い返し**
   - (c) 新しい**改善アクション案**
4. **人が確認・確定**：
   - (a) の自動判定を確認し、違えば**上書き**できる（人が最終責任）
   - (c) のアクション案を採用/編集/却下 → 採用分を今スプリントのアクションとして保存
   - このスプリントの結果（判定・問い返し・確定アクション）を**ナレッジに蓄積**
5. 次スプリントで 1 に戻る＝**ループが閉じる**（保存したアクションが次回の自動判定対象になる）

### 5.2 効果追跡ループ（この製品の核）
```
[スプリントN]  KPT → AI問い返し → アクション確定(保存) → ナレッジ蓄積
                                        │ 繰り越し
                                        ▼
[スプリントN+1] KPT → AI問い返し
                     ├─ 前回アクションを自動判定(効いた/効いてない/未着手＋理由)  ← 人は上書き可
                     ├─ 問い返し（前回が効いてないなら「なぜ形骸化した?」まで踏み込む）
                     └─ 新アクション案 → 確定 → ナレッジ蓄積
```
- **自動判定の根拠**：当スプリントの付箋（同じProblemが再発してないか / Keepに改善の言及があるか等）＋ これまでのナレッジ から、AIが前回アクションの効果を判定する。**付箋に手がかりが無く判定できないものは、AIがユーザに問い返して回答をもらう**。AI判定は**人が上書き可**。
- **アクションの寿命**：［要レビュー］「効いてない/未着手」の扱い＝AIが「継続/見直し」を提案し、**人が繰り越すか打ち切るかを最終決定**（全自動で繰り越さない）。「効いた」は完了扱い。
- **ナレッジ**：チーム単位で、確定アクション・その判定・問い返しを貯め続け、毎回のAI入力に渡す（＝過去の学びを踏まえて判定・問い返しの精度を上げる。差別化の核）。

## 7. 私が置いた前提の一覧（＝要レビューまとめ）

全項目 相原さん確定済（★）。

1. ★ **スプリント＝ふりかえり1回**で固定（1スプリントに複数回のふりかえりはしない）
2. ★ **効果判定のタイミング**：次スプリントのふりかえりで行う（改善は次スプリントを回さないと分からないため）
3. ★ **AI出力の確定方法**：アクション案は人が採用/編集/却下して保存（AIが直接DBに書かず人が承認）
4. ★ **前回アクションの効果は AI が自動判定**する（効いた/効いてない/未着手＋理由）。人は上書き可
5. ★ **ナレッジ蓄積**：これまでのアクション・判定・問い返しをチーム単位で貯め、毎回のAI入力に渡す
6. ★ **自動判定の根拠**：AIは**付箋**から判定する。付箋に効果への言及が無く判定できないアクションは、**AIがユーザに問い返して回答をもらう**
7. ★ **効いてない/未着手アクションの繰り越し**：AIが継続/見直しを提案し、**人が繰り越すか打ち切るか最終決定**（全自動繰り越しはしない）
8. ★ **ナレッジの持ち方**：MVPは履歴をDB保持し関連分をAIに渡す。AI要約“ダイジェスト”はv1
9. ★ **判定の値**：効いた / 効いてない / 未着手 の3値（細かくしない）。＋AIが判定できず**確認中**の状態を持つ

## 8. MVP完了の定義（受け入れ観点）

- ログインして自分のチームのふりかえりだけ見られる
- スプリントを作り、KPT付箋をCRUDできる
- ボタン1つでAI問い返し（前回アクションの自動判定＋問い返し＋新アクション案）が構造化で表示される
- AI案を採用してアクション保存でき、**次スプリントでAIが前回アクションを自動判定し、人が確認/上書きできる**（＝ループが1周見える）
- これまでの結果がナレッジとして貯まり、次のAI入力に渡る
- 本番URLで動く
