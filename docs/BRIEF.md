## 0. これは何か（一言）

転職ポートフォリオ兼スキル補完のために、**AI駆動で個人開発するWebアプリ**。
テーマは「開発プロセス支援系」。作るのは **AIふりかえり（レトロスペクティブ）支援ツール**。

## 3. 差別化ポイント（ここがブレたら意味がない）

競合（Parabol / EasyRetro / Retrium / TeamRetro / Neatro / Echometer）を調査済み。結論：

- **AI機能はもう"あって当たり前"**。ただし各社のAIは「付箋を要約・グルーピングする」＝**過去の整理**止まり。
- **誰も埋めてない空白**：
  1. 「そのアクション、本当に再発防止になってる？形骸化してない？」と**問い返す**役
  2. 出したアクションが**次スプリントで効いたかを追跡する閉ループ**
- スプリント見積り系（PlanningPokerAI, SprintPoker 等）は**AI見積りが既にコモディティ化**しており、個人開発では分が悪い → **やらない**。

→ 差別化を「**改善ループの深さ**」に置く。3本柱：
1. AIを「まとめ役」ではなく「**問い返す先輩スクラムマスター**」にする（作者の実績＝参入障壁）
2. **アクションの効果追跡ループを閉じる**（ふりかえり→アクション→次スプリントで効果検証）
3. 日本のアジャイル現場文脈に寄せる（※補助的。主軸は1・2）

## 4. MVPスコープ（確定）

方針：**差別化の核だけは削らない・それ以外は徹底的に削る**。

### MVPに入れるもの（これで本番公開する）
1. **最小認証**（メール or GitHubログイン1つだけ）
2. **チーム／スプリント作成**（階層は浅く：チーム＞スプリントだけ）
3. **付箋CRUD**（KPT形式・**リスト表示でOK**。追加/編集/削除できれば十分）
4. **AI問い返し（★差別化の核・絶対入れる）**：ボタン押下 → 付箋をまとめてClaude APIに投げ、**改善アクション案＋「その再発防止、形骸化してない?」の問い返し**を構造化で返して表示
5. **アクション化＋前回アクションの持ち越し表示**：問い返しで出たアクションを保存 → 次スプリント画面に前回アクションを出して「効いた/効いてない」をチェックできるだけ（＝効果追跡ループの芽を最小コストで見せる）
6. **本番URLで公開デプロイ**

### MVPでやらないこと（v1以降）
- ドラッグ&ドロップの付箋UI（リストで代替）
- リアルタイム同時編集
- メトリクス連携・グラフ（効果追跡の"完全版"はv1）
- CSV取込・外部連携（Jira等）
- 複数チームの権限管理・招待フロー
- モバイル最適化
- テンプレート多数（KPT 1つでいい）

削る判断軸：「**それが無いと"問い返して改善を追う"という差別化が成立しないか?**」でYesだけ残した。

## 5. AI駆動開発の"型"（意識的に練習する＝転職で語る中身）

1. 要件・画面・データモデルは**自分で設計**（＝EM/設計力の見せ場、人間主導）
2. Claude Code に**段階的に実装**させる（小さいPR単位。丸投げしない）
3. 生成コードを**レビュー**し、なぜその実装か説明できる状態にする
4. **テストを書かせる/追加**（作者は自動テスト導入の実績あり＝強み）
5. 週末に**デプロイ**、実際に使って次の課題を出す

週次リズム：平日夜に分割で設計＋実装、週末にデプロイ＆ふりかえり。詰まった技術（App Routerのデータ取得、認証、ORM 等）はそこだけ公式doc等で just-in-time に補う。
